# Prompts — Nanobanana

## 1. Validação de Mercado (rodar antes de criar oferta)

```
Preciso validar se um produto/nicho tem demanda real no Brasil para venda via native ads (Taboola/Outbrain).

Produto: [DESCREVER PRODUTO]
Ticket médio: R$ [VALOR]
Público-alvo: [DESCREVER]

Analise:
- Tamanho do mercado no Brasil (em USD ou BRL, com fonte)
- Taxa de crescimento anual (CAGR)
- Principais players nacionais e market share
- Volume de importação do produto no Brasil
- Tendências de busca no Google Trends (últimos 5 anos)
- Presença do nicho em redes sociais (Instagram, TikTok, Pinterest — perfis grandes, hashtags)
- Se existe consumo aspiracional da classe B/C nesse segmento
- Ticket médio praticado no mercado vs. meu ticket proposto
- Riscos: compliance com plataformas de native ads, sazonalidade, concorrência

Formato: dados objetivos com fontes. Sem enrolação.
```

## 2. Análise de Público — Native Ads Brasil

```

Mapeie:

- Comportamento de consumo online: o que compram por impulso, ticket tolerável
- Gatilhos emocionais que funcionam nesse público (escassez, oportunidade, status, vingança de preço, descoberta)

- Objeções comuns desse público antes de comprar online
- Formato preferido: vídeo, texto longo, galeria de fotos, quiz

Seja específico para o mercado brasileiro. Não quero dados genéricos dos EUA.
```

## 3. Geração de Ângulos para Advertorial

```

Preço: R$ [VALOR] (preço original: R$ [VALOR])
Narrativa atual: [DESCREVER — ex: "liquidação de importador que perdeu contrato com grifes"]

Gere 10 ângulos diferentes para o advertorial. Cada ângulo deve ter:
- Nome do ângulo (2-3 palavras)
- Premissa editorial (1 frase — qual "notícia" ou "descoberta" está sendo contada)
- Headline do Taboola (máx. 60 caracteres, estilo editorial/curiosidade)
- Gancho emocional principal (escassez, oportunidade, revelação, status, economia)

- Deve funcionar para público feminino 30-55, classe B/C, Brasil

## 4. Estrutura de Advertorial Completo

```
Escreva um advertorial completo em HTML para o seguinte produto:

Produto: [DESCREVER]
Ângulo: [DESCREVER — ex: "Importador perde contrato milionário e vende porcelana de grife a preço de custo"]
Público: [DESCREVER]
CTA: [URL DA PÁGINA DE VENDAS]

Estrutura obrigatória:
1. Headline editorial (parece matéria de portal)
2. Subtítulo com contexto/localização
3. Foto hero com legenda estilo jornalístico
4. Parágrafo de abertura: gancho noticioso, o que aconteceu
5. Contexto: por que isso é relevante agora
6. Prova social: números, depoimentos, fotos
7. Explicação do produto/oferta (sem parecer venda)
8. Objeções respondidas naturalmente no texto
9. Escassez real (estoque, prazo, limite por CPF)
10. CTA principal (botão)
11. Seção de comentários fake (6-8 comentários variados: pergunta, resposta, elogio, dúvida sobre entrega, comparação com marca cara)
12. CTA final

Regras de copy:
- Tom jornalístico no início, vai ficando mais pessoal no meio
- Frases curtas. Parágrafos de 2-3 linhas máximo
- Zero jargão de marketing ("imperdível", "incrível", "fantástico")
- Usar números específicos, não genéricos
- Não usar travessão (—) em excesso
- Linguagem natural brasileira, sem parecer tradução nem IA
- Incluir pelo menos uma frase que gere FOMO real
```

## 5. Headlines para Taboola (batch)

```
Gere 20 headlines para Taboola ads com as seguintes restrições:

Produto: [DESCREVER]
Ângulo: [DESCREVER]
Público: mulheres 30-55, classe B/C, Brasil

Regras do Taboola:
- Máximo 60 caracteres (OBRIGATÓRIO — contar cada headline)
- Sem pontuação excessiva (máx. 1 ponto de exclamação por headline)
- Sem ALL CAPS (exceto siglas)
- Sem promessas de saúde/renda
- Sem "você não vai acreditar" e variações batidas

Formatos a cobrir (pelo menos 2 de cada):
- Curiosidade: "Por que [X] está [Y]?"
- Revelação: "[Profissão/grupo] revela o que [ação]"
- Localização: "[Cidade/Estado]: [fato surpreendente]"
- Temporal: "Até [data], [oportunidade]"
- Social proof: "[Número] [pessoas] já [ação]"

Para cada headline, indique:
- Headline
- Contagem de caracteres
- Formato usado
- Sugestão de thumbnail (1 frase)
```

## 6. Review Anti-IA de Copy

```
Revise o texto abaixo e identifique todos os padrões que denunciam escrita por IA:

[COLAR TEXTO]

Marque especificamente:
- Uso excessivo de travessão (—)
- Fórmulas repetitivas ("ideal para", "perfeito para", "Para quem")
- Adjetivos genéricos empilhados ("sofisticado e elegante")
- Estrutura muito simétrica entre itens (todos com mesmo formato)
- Conectivos formais demais pro contexto ("Ademais", "Outrossim", "Nesse sentido")
- Tom professoral ou enciclopédico
- Frases que ninguém falaria em conversa real

Depois reescreva o texto corrigindo cada problema. O tom deve ser: empresa séria mas acessível. Não é blog, não é redação do ENEM, não é ChatGPT. É copy de e-commerce brasileiro real.
```

## 7. Comentários Fake Realistas

```
Gere [N] comentários fake para a seção de reviews de uma página de vendas.

Produto: [DESCREVER]
Preço: R$ [VALOR]
Opções de escolha: [LISTAR — ex: "Serviço Essencial / Família / Completo"]
Público: [DESCREVER]

Tipos obrigatórios:
- 2 elogios de quem recebeu (mencionar opção que escolheu, detalhe físico do produto)
- 1 pergunta de quem ainda não comprou (dúvida real)
- 1 resposta de outro comprador ajudando
- 1 sobre velocidade de entrega
- 1 comparando com marca/loja cara
- 1 do suporte respondendo dúvida (tom profissional mas humano)

Regras:
- Nomes brasileiros reais e variados (não só "Maria" e "João")
- Erros de digitação ocasionais, abreviações (vc, pq, kk, tbm)
- Tamanhos variados (alguns curtos, alguns 2-3 linhas)
- Nunca usar travessão (—)
- Nenhum comentário deve parecer template ou ter a mesma estrutura que outro
- Incluir likes variados (0 a 12) e tempos variados ("5 min" a "2 dias")
```

Nicho: [DESCREVER]
Produto: [DESCREVER]

```
